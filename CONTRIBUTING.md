# Contributing to ZeroLeak DevTools

Thanks for your interest in contributing! This project is meant to stay simple, secure, and easy to use.

## How to contribute

1. Fork the repository.
2. Create a branch for your change:
   - `git checkout -b feature/your-feature`
   - `git checkout -b fix/your-bug`
3. Keep changes small and targeted.
4. Add a short summary of your change in the PR description.
5. Submit a pull request.

## Good first contributions

- Add more offline tools
- Improve the GraphQL linting and validation logic
- Add better error messages for JSON and YAML handling
- Improve the UI or accessibility
- Add browser compatibility notes

## Project priorities

- Offline-only behavior
- No network usage
- No persistent storage of input
- Simple, readable code
- Beginner-friendly documentation
